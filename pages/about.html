Project Information

About This Project

LibraryMS is a simple web application created to manage books, members, and loan records through one clear and user-friendly interface.

Overview

What this system is designed to do

The project demonstrates how a frontend website can connect to a relational database and support day-to-day library tasks.

Purpose

Built for practical library management

The main goal of the system is to organise library data in one place. It allows users to view records, keep data structured, and support common workflows such as managing books, members, and borrowing activity.

Database

Structured around three core tables

  • Books stores book information and copy availability
  • Members stores registered user details
  • Loans stores issue, return, and overdue records

Main Features

Key functions of the system

  • dashboard summary for books, members, and loans
  • book record management
  • member CRUD operations and search
  • loan issue, return, and overdue tracking
  • ERD and SQL setup for database support

Technology

Tools used in development

  • HTML, CSS, and JavaScript for the frontend
  • Supabase and PostgreSQL for the database
  • GitHub for teamwork and version control
  • Mermaid for the ERD visualisation

Workflow

How the system works in practice

Books

Users can review book records, update information, and manage available copies for the library collection.

Members

The members page helps store and maintain user information in a clear and searchable format.

Loans

The loans area records borrowing activity, supports returns, and highlights overdue items that need attention.

Team Roles

How the work was divided

  • Project coordination and home page development
  • Database design, SQL setup, and ERD creation
  • Books page development
  • Members and loans page development

Summary

Current stage of the project

At this stage, the system includes a connected dashboard, database structure, and working pages for books, members, and loans. The project focuses on clear structure, accessibility, and practical database interaction.
